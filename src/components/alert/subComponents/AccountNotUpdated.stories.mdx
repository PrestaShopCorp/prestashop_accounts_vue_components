# Alert block: PS Accounts module is not up to date

When the PS Accounts module is not installed in the module manager of the backoffice, the
linking process cannot be done properly. So the user should be warned with that message
and the process is blocked until the module is installed again.

<Story name='Not installed' height='220px'>
  {{
    components: {AccountNotUpdated},
    props: {
      accountIsUptodate: {
        default: boolean('accountIsUptodate', false),
      },
    },
    methods: {
      event: action('update'),
    },
    template: '<AccountNotUpdated class="m-4" :accountIsUptodate="accountIsUptodate" @install="event" />',
  }}
</Story>

# Integration

_Notice: If you need to integrate the standard behavior of PS Accounts, please see the
[`PsAccounts` component in the integration section](?path=/docs/integration-psaccounts-panel--connected)._

If you need a custom integration of PS Accounts sub-components, here is the AccountNotInstalled block:

```html
<AccountNotUpdated :accountIsUptodate="accountIsUptodate" @install="event" class="m-4" />
```
